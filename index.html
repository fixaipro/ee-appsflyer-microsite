// 02 · ATT BUSINESS CASE

The ATT Imperative

With 55% iOS opt-out, is ATT worth pursuing? Yes — the downstream value far outweighs deployment friction. iOS drives 69% of Telco UK app revenue and is non-negotiable.

OPT+
ATT Opt-In: Deterministic Attribution
Real-time IDFA enables full campaign reporting broken by media strategy, format, and creative. Unlocks ML/LLM models for NBA prediction, fraud prevention, LTV cohorts, and in-platform bid optimization across PDP, PCP, Add-to-Cart, Revenue and Checkout events.
45% CONSENT RATE
OPT−
ATT Opt-Out: Aggregated Only
Without IDFA: only revenue event available, campaign-level aggregated reports with no breakdowns by media strategy or format. Blind spots in Retargeting, fraud detection severely hampered, LTV prediction degraded.
55% OPT-OUT TODAY
SKAN
SKAN: iOS Attribution Beyond the ATT Wall
SKAN conversion schema templates provide aggregated iOS attribution without IDFA. Industry templates available for Telecoms. Complements ATT — not a replacement. Hybrid ATT + SKAN approach is Telco UK's optimal measurement posture.

DIRECT DEEP LINK

App Already Installed

Opens the app and routes directly to a specific in-app activity using URI scheme or Universal/App Links. OneLink data is passed via UDL API and the DeepLink object carries deep_link_value.

DEFERRED DEEP LINK

App Not Installed

Redirects to correct App Store. After install, the app opens and AppsFlyer SDK fires the UDL API or GCD callback to retrieve original click parameters and route user to intended destination.

BRANDED DOMAIN

Telco UK CNAME Setup Required

DNS admin configures CNAME: app.telcoprovider.co.uk → telcoprovider-app.customlinks.appsflyer.com. Provides branded link experience with full attribution capability.

UNIVERSAL LINKS

Apple's Preferred Method (iOS 9+)

HTTP/HTTPS links that iOS intercepts and opens directly in the app — no browser redirect. Requires Associated Domains entitlement and AASA file hosted by AppsFlyer on the OneLink subdomain.

URI SCHEME

Fallback Mechanism

telco://mainactivity style URLs. Used as fallback when Universal Links fail. Uniqueness not enforced — disambiguation dialog may appear if multiple apps claim same scheme.

// 04 · DEFERRED DEEP LINKING

UDL vs GCD

Two methods, two scenarios. Both must be implemented. UDL for owned channels and most media. GCD as the critical fallback for SRN traffic (Google, Meta, TikTok, Snapchat, Amazon).

UDL
Unified Deep Linking
// OWNED CHANNELS · NON-SRN MEDIA
  • Crucial for email, push notifications, owned web, QR codes
  • Works for paid media excluding Google, Meta, Snap, TikTok, Amazon
  • Returns: deep_link_value + deep_link_sub1-10
  • Android: onDeepLinking() in DeepLinkingListener
  • iOS: didResolveDeepLink() in DeepLinkDelegate
  • Flutter: appsflyerSdk.onDeepLinking() callback
if status == FOUND && deep_link_value:
  route(deep_link_value)
elif status == NOT_FOUND:
  // fallback to GCD
  onConversionDataSuccess(data)
GCD
Get Conversion Data
// SRN MEDIA · UDL FALLBACK
  • Critical for Google, Meta, Snap, TikTok, Amazon (SRNs)
  • Activates when UDL returns NOT_FOUND or click-to-install exceeds 15 mins
  • Returns: media_source, campaign, af_status, click_time, install_time, af_sub1-5
  • onConversionDataSuccess callback in both Android and iOS
  • Check is_first_launch: true + af_status: Non-organic before routing
onConversionDataSuccess(data):
  if data.af_status != "Non-organic": stop
  if data.is_first_launch != true: stop
  if data contains deep_link_value: stop
  elif data contains legacy_param:
    route(mapToDeepLinkObject(data))
// CRITICAL NOTE FROM IMPLEMENTATION GUIDE

It is recommended and required to configure both UDL and GCD. UDL handles direct and non-SRN deferred scenarios. GCD is the essential fallback for SRN traffic and when click-to-install exceeds the 15-minute UDL matching window. Omitting either method will result in broken deep linking experiences for a significant portion of Telco UK's user journeys.
